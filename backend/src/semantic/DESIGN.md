# Semantic 模块设计 (Semantic Module Design)

`semantic` 模块在元 AST 的基础上构建多层次的语义图谱，提供代码导航、重构和深度分析能力。

## 1. 核心功能

### 1.1 Scope Graph 构建
- **语言无关引用解析**: 通过分析元 AST 中的定义与使用节点，构建 Scope Graph。
- **跨文件解析**: 处理模块导入导出，建立项目级的符号引用网。

### 1.2 语义分析
- **符号定位**: 提供跳转到定义、查找引用、类型跳转等功能。
- **重构支持**: 提供重命名、提取函数等操作的语义安全验证。

## 2. 核心组件
- `GraphBuilder`: 从元 AST 提取语义关系并填充图谱。
- `SymbolResolver`: 执行符号查找与路径解析。
- `RefactorEngine`: 负责生成语义化的变更请求（Change Request）。

## 3. 设计原则
- **基于元 AST**: 严禁直接依赖具体的语法解析器（如 Tree-sitter）。
- **增量更新**: 随着 Thread 中的变更，图谱能够进行增量局部更新。
