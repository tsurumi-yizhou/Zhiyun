# Knowledge 模块设计 (Knowledge Design)

`knowledge` 模块负责管理项目的非结构化与半结构化知识，通过 RAG (Retrieval-Augmented Generation) 增强 Agent 的决策能力。

## 1. 核心功能

### 1.1 RAG 流程
- **混合检索**: 结合向量数据库的语义搜索与知识图谱的逻辑搜索。
- **上下文压缩**: 自动识别并修剪与当前任务无关的冗余信息。

### 1.2 知识图谱
- **架构感知**: 记录设计模式、业务逻辑关系和架构决策记录 (ADR)。
- **逻辑推理**: 支持复杂的图查询，如“查找所有受此架构变更影响的模块”。

## 2. 核心组件
- `VectorStore`: 存储代码片段、文档和注释的嵌入向量。
- `KnowledgeGraph`: 维护项目的高层架构关系。
- `Retriever`: 执行多模态检索与重排 (Reranking)。

## 3. 设计原则
- **事实优先**: 确保 Agent 检索到的信息是基于当前代码库的真实状态。
- **动态更新**: 随着代码的修改，索引和图谱应能实时或准实时更新。
