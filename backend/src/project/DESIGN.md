# Project 模块设计 (Project Design)

`project` 模块负责管理代码的工程上下文，包括目录结构、构建系统和依赖关系，而不假设具体的编程语言。

## 1. 核心功能

### 1.1 构建系统抽象
- **通用任务接口**: 定义统一的 `Build`, `Test`, `Run` 任务接口。
- **工具适配**: 通过插件支持 `Cargo`, `npm`, `Bazel`, `CMake` 等多种构建工具。

### 1.2 依赖管理
- **依赖图解析**: 提取项目依赖关系，为 `knowledge` 模块提供上下文。
- **环境隔离**: 管理项目所需的运行环境（如虚拟环境、容器路径）。

## 2. 核心组件
- `WorkspaceManager`: 识别项目根目录与多包 (Monorepo) 结构。
- `BuildSystemAdapter`: 桥接具体的构建工具。
- `DependencyResolver`: 分析依赖树并检测冲突。

## 3. 设计原则
- **结构化工具**: 所有的项目操作都必须通过结构化的工具调用，严禁直接执行非受控的 Shell 命令。
- **配置驱动**: 通过解析项目配置文件（如 `package.json`, `Cargo.toml`）自动推断项目类型。
