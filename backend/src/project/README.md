# Project 模块 (Project Management)

`project` 模块负责管理代码的工程上下文，包括目录结构、构建系统和依赖关系，而不假设具体的编程语言。

## 核心功能

- **构建系统抽象**: 定义统一的 `Build`, `Test`, `Run` 任务接口。
- **依赖管理**: 提取项目依赖关系，为 `knowledge` 模块提供上下文。
- **环境隔离**: 管理项目所需的运行环境（如虚拟环境、容器路径）。

## 核心组件

- [manager.rs](./manager.rs): `ProjectManager` 管理项目目录结构与配置。
- [config.rs](./config.rs): `ConfigLoader` 加载与合并项目配置。
- [dependency.rs](./dependency.rs): `DependencyManager` 管理项目依赖关系与版本。

## 设计原则

- **结构化工具**: 所有的项目操作都必须通过结构化的工具调用，严禁直接执行非受控的 Shell 命令。
- **配置驱动**: 通过解析项目配置文件（如 `package.json`, `Cargo.toml`）自动推断项目类型。

## 架构设计细节

### 1. 通用构建抽象
- **任务接口**: 抽象了 `Build`, `Test`, `Run` 等标准工程任务，屏蔽了不同构建系统的命令差异。
- **适配器模式**: 通过 `BuildSystemAdapter` 灵活扩展对 `Cargo`, `npm`, `Bazel` 等工具的支持。

### 2. 工程上下文感知
- **依赖图谱**: 自动解析并导出项目的完整依赖树，为语义分析和知识检索提供宏观背景。
- **Monorepo 支持**: `WorkspaceManager` 能够智能识别多包结构，支持跨包的代码操作。

### 3. 受控执行环境
- **工具安全**: 强制要求所有工程操作通过结构化工具调用，避免了直接执行任意 Shell 命令带来的安全隐患。
- **配置驱动**: 采用零配置设计理念，通过分析项目元数据文件自动配置运行参数。
